发表于 2019-07-11 | 分类于 设计模式 | | 阅读次数:

模板方法模式（Template Method Pattern）: 定义一个操作中算法的框架，而将一些步骤延迟到子类中。使得子类可以不改变一个算法的结构即可重定义算法的某些特定步骤。

模板方法模式
优势：

  • 封装不变部分，扩展可变部分
  • 提取公共部分代码，便于维护
  • 行为由父类控制，子类实现

劣势：

  • 阅读有点难度

使用场景：

  • 有能提取的公共方法
  • 重要复杂的算法
  • 需要大量扩展的方法类
